## Hardware Architecture

This device hosts two independent NPU compute paths. **The chip roles are the reverse of what the names suggest:**

| Component | Chip | Interface | Services |
|---|---|---|---|
| **Host SoC** | Rockchip RK3588 (4×A76 + 4×A55) | On-die | RKNPU3 (6 TOPS INT8, 3×2 TOPS) — **Embedding only** via uvicorn port 18080 |
| **NPU Co-processor** | Rockchip **RK1828** (PCIe, Product RM1828SA0-F) | PCIe 2.1 ×1 lane, 5 GT/s · `0003:31:00.0` | **LLM** via rkllm3-server port 18001; **ASR** port 18002; **TTS** port 18003 |

Source of truth: `/userdata/model_hub/model_hub_config.yaml` on device + active process inspection (2026-06-20). Product serial: R1BCA260200353.

**Chip naming note:** The product is an RK1828 (SO-DIMM module RM1828SA0-F). The RKNN3 SDK v0.3.0b0 release notes referenced "RK1820" — this refers to the RK182X chip family; our specific device is the RK1828 variant. Model IDs in this repo use `-rk1820` suffix (historical, kept for consistency).

Models and benchmarks are tagged to the specific chip that runs them.

### RK3588 RKNPU3 (on-die)

| Spec | Official | Measured | Assessment |
|---|---|---|---|
| NPU peak | **6 TOPS INT8** (3×2 TOPS) | — | Used for embedding only in this setup |
| Embedding hit@1 | — | **1.000** (3-seed 2026-06-25) | ✅ Perfect retrieval |
| Embedding p50 | — | **143 ms** | Acceptable for RAG use |

**Key insight — why high TPS despite PCIe bottleneck:** The RK1828's 3D-stacked DRAM (1 TB/s internal bandwidth) keeps all model weights on-chip. The PCIe 2.1 ×1 link (~500 MB/s) only transfers input tokens and output activations — not the weight matrix on every token. This architecture achieves server-class throughput at edge power levels.

**Key insight — conversation_drift FAIL:** The 21.19% structured-extraction drop over 20 turns is a **context-length constraint** (768 token RKLLM hard limit), not a chip capability issue. The RK1828's 20 TOPS / 1 TB/s DRAM is fully capable of longer context; the limitation is in the current RKLLM runtime firmware.

## Known Limitations

- **Context limit 768 tokens** — Hard limit imposed by the RKLLM runtime; effectively limits response length and prevents few-shot benchmarks.
- **NPU orphan process lock-up** — A stuck NPU inference process (rknn3_transfer_proxy) can lock the NPU hardware; recovery requires reboot. No software workaround yet.
- **HuggingFace not reachable** — The device cannot access huggingface.co directly. Only pre-cached datasets work. GSM8K (openai/gsm8k) is cached; MMLU and HellaSwag are not.
- **Two-chip architecture** — LLM/ASR/TTS run on the RK1820 PCIe NPU (port 18001/18002/18003); Embedding runs on the RK3588 internal RKNPU3 (port 18080). Performance and availability of each are independent.
- **`qwen3-vl-2b` conversation_drift FAIL (2026-06-26)** — structured_extraction quality drops 21.19% over 20 turns (pos0=0.95→pos20=0.74), exceeding the 15% threshold. The model degrades on structured response extraction in long conversations. TTFT/throughput/translation remain PASS. ctx=768 token limit likely contributes (history fills context, degrading output structure).
- **RKNN3 Model Zoo models PENDING-VERIFY** — Full `v1.0.4` LLM/VLM/OCR-relevant coverage from Lenovo Box `/RKNN3_SDK/rknn3_models` is registered in `models.yaml` as of 2026-06-29. Entries still require per-model service loading and E2E calibration. CNN example models were scanned but intentionally excluded from the LLM/VLM benchmark matrix.
- **ASR 3-seed PASS confirmed 2026-06-24** — CER 4.73%±0 (zero variance), RTF 0.0768±0.0045, p50 319ms (5 languages: zh/en/ja/ko/yue). zh CER inflated by Traditional/Simplified charset mismatch; acoustic accuracy near 0% with normalization.

**核心洞察 — 为何 PCIe 接口窄却 TPS 高：** RK1828 通过 3D 堆叠 DRAM（内部 1 TB/s 带宽）将模型权重全程留在片内。PCIe 2.1 ×1（~500 MB/s）只传输输入 token 和输出激活值，不传权重矩阵。这一架构在边缘功耗下实现了接近服务器级的吞吐。

**核心洞察 — conversation_drift FAIL 的本质：** 20 轮后结构化提取下降 21.19% 是 **RKLLM runtime 的 768 token 上下文硬性限制**，不是芯片算力问题。RK1828 的 20 TOPS + 1 TB/s DRAM 完全有能力支持更长上下文；制约来自当前 RKLLM 固件。

### 已知局限

- **上下文上限 768 token** — RKLLM runtime 硬性限制，无法运行 few-shot benchmark，也限制响应长度。
- **NPU 孤儿进程锁死** — `rknn3_transfer_proxy` 卡死后会锁死 NPU 硬件，需 reboot 恢复。
- **HuggingFace 不可达** — 设备无法访问 huggingface.co，只有预缓存数据集可用（GSM8K 已缓存；MMLU、HellaSwag 未缓存）。
- **双芯片架构** — LLM/ASR/TTS 在 RK1820 PCIe NPU 上（port 18001/18002/18003）；embedding 在 RK3588 RKNPU3 上（port 18080）。
- **RKNN3 Model Zoo 模型待验证** — 官方 TPS 数据未经本 harness E2E 测试校准。
- **ASR 3-seed PASS（2026-06-24）** — CER 4.73%±0（零方差），RTF 0.0768±0.0045，p50 319ms（5 语言：zh/en/ja/ko/yue）；zh CER 因繁简字集差异偏高，声学转写正确率接近 0%。
- **qwen3-vl-2b 对话漂移 FAIL（2026-06-26）** — 结构化提取质量随对话轮次下降：第 0 轮 0.95 → 第 20 轮 0.74，max_drop=21.19%，超过 15% 阈值；ctx=768 token 限制是主因（历史记录填满上下文，导致结构化输出质量下降）。TTFT/吞吐/翻译维度不受影响。
