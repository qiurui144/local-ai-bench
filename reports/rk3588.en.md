## Hardware Architecture

This device hosts two independent NPU compute paths. **The chip roles are the reverse of what the names suggest:**

| Component | Chip | Interface | Services |
|---|---|---|---|
| **Host SoC** | Rockchip RK3588 (4×A76 + 4×A55) | On-die | RKNPU3 (3×2 TOPS) — **Embedding only** via uvicorn port 18080 |
| **NPU Co-processor** | Rockchip **RK1820** (PCIe, Device 182a) | PCIe `0003:31:00.0` | **LLM** via rkllm3-server port 18001; **ASR** port 18002; **TTS** port 18003 |

Source of truth: `/userdata/model_hub/model_hub_config.yaml` on device + active process inspection (2026-06-20).  
Note: the device's system calls the RK1820 "rk1828" internally, but the official SDK name is **RK1820** (RKNN3 SDK V0.3.0b0, released 2025-09-12).

Models and benchmarks are tagged to the specific chip that runs them.

## Skipped Dimensions

The following dimensions are permanently skipped for `qwen3-vl-2b-rk1820`:

| Dimension | Reason |
|---|---|
| `general_ability` | ctx=768 token limit prevents few-shot GSM8K; MMLU/HellaSwag unavailable (HuggingFace not reachable from device, cache incomplete). Reference: GSM8K informal run 66% (2026-06-19, non-calibrated). |
| `prefill_decode` | Low sample count unreliable at ctx=768 |
| `conditioned` | ctx=768 insufficient for multi-turn system prompts |
| `scenarios` | ctx=768 insufficient; requires conditioned to pass first |
| `accuracy` | VLM visual grounding not supported by text-only task_type |
| `stability`, `concurrency` | Long-duration tests impractical on NPU with orphan lock risk |
| `embedding`, `rerank`, `asr`, `ocr` | Wrong capability class for this model |

## Known Limitations

- **Context limit 768 tokens** — Hard limit imposed by the RKLLM runtime; effectively limits response length and prevents few-shot benchmarks.
- **NPU orphan process lock-up** — A stuck NPU inference process (rknn3_transfer_proxy) can lock the NPU hardware; recovery requires reboot. No software workaround yet.
- **HuggingFace not reachable** — The device cannot access huggingface.co directly. Only pre-cached datasets work. GSM8K (openai/gsm8k) is cached; MMLU and HellaSwag are not.
- **Two-chip architecture** — LLM/ASR/TTS run on the RK1820 PCIe NPU (port 18001/18002/18003); Embedding runs on the RK3588 internal RKNPU3 (port 18080). Performance and availability of each are independent.
- **RKNN3 Model Zoo models PENDING-VERIFY** — Qwen2.5-0.5B/3B, InternVL3-2B, FastVLM-1.5B, Qwen2.5-VL-3B have official TPS data but have not yet been run through the benchmark harness. Pre-converted models available at https://console.box.lenovo.com/l/H1fig1 (code: rknn).
- **ASR 3-seed PASS confirmed 2026-06-24** — CER 4.73%±0 (zero variance), RTF 0.0768±0.0045, p50 319ms (5 languages: zh/en/ja/ko/yue). zh CER inflated by Traditional/Simplified charset mismatch; acoustic accuracy near 0% with normalization.

### 已知局限

- **上下文上限 768 token** — RKLLM runtime 硬性限制，无法运行 few-shot benchmark，也限制响应长度。
- **NPU 孤儿进程锁死** — `rknn3_transfer_proxy` 卡死后会锁死 NPU 硬件，需 reboot 恢复。
- **HuggingFace 不可达** — 设备无法访问 huggingface.co，只有预缓存数据集可用（GSM8K 已缓存；MMLU、HellaSwag 未缓存）。
- **双芯片架构** — LLM/ASR/TTS 在 RK1820 PCIe NPU 上（port 18001/18002/18003）；embedding 在 RK3588 RKNPU3 上（port 18080）。
- **RKNN3 Model Zoo 模型待验证** — 官方 TPS 数据未经本 harness E2E 测试校准。
- **ASR CER 已校准** — CER 4.73%（5 语言：zh/en/ja/ko/yue），RTF 0.084（2026-06-20）；zh CER 因繁简字集差异偏高，声学转写正确率接近 0%。
